Rava

Methods

Rava has two methods, the first is the decorate method which allows you to assign a rava configuration object to an element based on it's css selector. The second is the utility method called query which allows you to do a search for child nodes of an element that matches a selector

decorate

signature

Rava.decorate(selector,ravaConfigObject)

parameters

selector

The first parameter for the decorate method is a css selector string. This is used to identify whether any element, either current or future, are elements that we should decorate.

Examples of valid selectors are;

  • "#nav li"
  • ".burger"
  • "rava-time"

ravaConfigObject

The default config Object is

{
    callbacks: {
        created:
        added:
        removed:
    },
    methods: {
        methodName: functionDefinition(){}
    },
    events:{
        eventName: functionDefinition(){},
        selector: {
            eventName : functionDefinition(){}
        }
    },
    data: {}
}
Callbacks

Callbacks are event listeners that are tied to the lifecycle of the configuration object.

EventCause
createdimmediately after the configuration object has been merged into the matched element
addedimmediately after the created callback occurs, and then every time the node element is attached to the dom tree
removedafter the element has been detached from the node tree
