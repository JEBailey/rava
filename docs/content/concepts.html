Concepts

binding

When Rava is instantiated it creates an empty object to maintain the mappings between the css selector and the associated configuration object.

So there is only one configuration tied to a sepecific selector. If you use the same selector with a different configuration, the new one will overwrite the old

The method to register a css string and it's configuration object is the bind method.

scope

All methods that are defined in the configuration object are both mapped to the element but they also have their scope bound to the element

What this means is that regardless of the context when the method is called, it's executed with the element as the this object
