References using Ref and Refs

A common use case is the need to create a reference to an object in the data object that is managed by a binding. To facillitate this there are two configurations that can be used. Ref is used to store a specific reference to data. The first object matching the selector will be stored. 
    rava.bind(".foo",{
        // stores a reference to the first element foo that is found as a child of 'this'
        // i.e. data.bar
        ref : {
            foo : "bar"
        },
    });
If there is only a string then ref with store 'this' to the data object. 
    rava.bind(".foo",{
        // stores a reference to 'this' as bar in the data object
        ref : "bar"
    });
By using an object in the configuration you can store multiple references 
    rava.bind(".foo",{
        ref : {
            ".foo" : "foo", // a single item of class foo
            ".bar" : "bar",
        }
    });
Refs is used if you need to store a collection of objects. Using Refs will also result in an array of one or more matching items 
    rava.bind(".foo",{
        refs : {
            ".foo" : "foo", // an array of items with the class of foo
            ".bar" : "bar", // an array of items with the class of bar
        }
    }
